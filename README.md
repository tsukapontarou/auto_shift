# Shift Auto

ローカルで開ける、アルバイトの週間シフト表アプリの最小版です。
現時点では、サンプルデータを使って曜日ごとの実時刻入力を定義済みシフトへ変換し、その内容を使って自動割り当てを実行できます。

## 現在できること

- アプリの見出しと説明を表示する
- サンプルスタッフ一覧を表示する
- 各スタッフの曜日別時刻入力を表示する
- 各スタッフの勤務可能時間と希望シフトを表示する
- スタッフごとに、曜日別の開始時刻 / 終了時刻を 30分刻みで入力する
- `timeBoundaries` がある場合は、隣接境界から固定シフトを自動生成する
- `timeBoundaries` がない場合は、従来の `slots / slotDefinitions` をそのまま使う
- 時刻入力を定義済みシフトへ自動変換し、3状態グリッドへ反映する
- スタッフを選んで、曜日 × 定義済みシフト の 3状態入力を編集する
- セルをクリックして `不可 → 可 → 希望` の順に切り替える
- 「希望を反映する」で、選択中スタッフの入力を現在データに反映する
- 1週間 × 定義済みシフト の必要人数を表で表示する
- 「シフト生成」ボタンで、更新後のデータに対して自動割り当てを実行する
- 各セルに必要人数、割り当てたスタッフ名、枠単位の不足、時間帯別不足を表示する
- 上部メッセージに、割り当て数、枠単位不足、時間帯別不足セル数を表示する
- シフト定義や必要人数に不整合がある場合、アプリを落とさずにエラーを表示する

## まだできないこと

- 複雑な最適化
- バックトラック探索
- スタッフ追加 / 削除や他項目の入力編集
- 1日に複数の時間帯を入力する機能
- 永続保存
- CSV / Excel 入出力
- 同日内の複数シフト制御
- 連勤制限
- 休憩制約
- 相性制約
- ログイン / 認証
- スタッフ専用画面への分離
- サーバー保存
- 複雑な権限管理
- 分単位の厳密な最適化
- 複数人の部分勤務を組み合わせて1枠を完全に埋める高度な探索

## 希望入力の状態

- `不可`
  - availability に入れない
  - preferences に入れない
- `可`
  - availability に入れる
  - preferences には入れない
- `希望`
  - availability に入れる
  - preferences に入れる

## 固定シフト定義

- `timeBoundaries` がある場合
  - 例: `["09:00", "11:00", "13:00", "15:00", "17:00", "20:00"]`
  - 隣接境界から `09:00-11:00`, `11:00-13:00`, `13:00-15:00`, `15:00-17:00`, `17:00-20:00` を自動生成する
- `timeBoundaries` がない場合
  - 既存の `slots` と `slotDefinitions` をそのまま使う

## 時刻入力から定義済みシフトへの変換

- 実際の入力可能範囲は、定義済みシフト全体の開始時刻から終了時刻まで
- 入力は `30分刻み`
- 開始 / 終了が両方未入力なら、その曜日は希望なしとして扱う
- 開始時刻が終了時刻以上ならエラーにし、その曜日は自動変換しない
- 入力時間帯と各固定シフトの重なりが `0分` なら `不可`
- 重なりが固定シフト長の `50% 未満` なら `可`
- 重なりが固定シフト長の `50% 以上` なら `希望`

## 時間帯別不足の計算

- 各固定シフトの内側を `30分刻み` で分割する
- 各30分バケットで、必要人数から実際にカバーしている人数を引いて不足人数を出す
- 不足人数が `0` の区間は表示しない
- 連続するバケットで不足人数が同じなら結合して表示する
- 不足がない場合は `時間帯別不足: なし` と表示する

## データ不整合の扱い

- 自動生成されたシフト名と `requirements` が一致しない場合は、該当セルを `定義エラー` として表示する
- この状態ではシフト生成は実行せず、上部メッセージに理由を表示する
- 画面は落とさず、希望入力や一覧表示は継続できる

## 現在の割り当てルール

- 貪欲法で、埋めやすい枠ではなく「候補者が少ない枠」から順に処理する
- 勤務不可の時間帯には割り当てない
- 同じ時間帯に同じスタッフを重複配置しない
- 週の最大勤務回数を超えない
- 希望シフトを優先する
- 週の最小勤務回数にまだ達していないスタッフを優先する
- すべて埋められない場合は、ルールを破らずに不足として残す

## ファイル構成

- `index.html`
  - 画面の骨組みとメッセージ領域
- `style.css`
  - 最小限の見た目、時刻入力、結果表示の装飾
- `sample-data.js`
  - スタッフ、時刻入力、`timeBoundaries` または固定シフト定義、必要人数のサンプルデータ
- `app.js`
  - サンプルデータの描画処理、固定シフト定義の解決、時刻入力の変換、希望入力の反映、自動割り当て処理

## 使い方

1. `C:\codex\shift-auto\index.html` をブラウザで開きます。
2. 「希望入力」セクションでスタッフを選び、曜日ごとの開始時刻 / 終了時刻を入力します。必要なら、その後で3状態グリッドを微調整します。
3. 「希望を反映する」を押してから、「シフト生成」を押すと、更新後の希望を使った結果と時間帯別不足が表に表示されます。サンプルデータでは `timeBoundaries` から 5 シフトを自動生成します。

## 補足

- 外部ライブラリは使っていません。
- `npm install` は不要です。
- ネットワークアクセスは使いません。
